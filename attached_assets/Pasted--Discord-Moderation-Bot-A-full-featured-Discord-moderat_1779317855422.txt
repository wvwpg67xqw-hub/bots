# Discord Moderation Bot

A full-featured Discord moderation and staff-network bot built with Discord.js v14. Includes 35 slash commands, SQLite persistence, role-based permissions, a web dashboard with Discord OAuth2 login, and an application system for server staff recruitment.

## Features

- **35 slash commands** covering warnings, bans, mutes, strikes, jail, promotions, and more
- **Application system** ‚Äî create custom forms, collect submissions, accept/deny applicants with automatic role assignment
- **Web dashboard** ‚Äî owner-only admin panel secured by Discord OAuth2
- **Role-based permissions** ‚Äî configure which roles can use each command, per server
- **SQLite database** ‚Äî zero-dependency persistent storage via Node's built-in `node:sqlite`
- **Health endpoint** ‚Äî Express server at `/health` for uptime monitoring (UptimeRobot etc.)

## Setup

### 1. Environment Variables

Set the following secrets in your environment:

| Variable | Required | Description |
|---|---|---|
| `TOKEN` | Yes | Discord bot token |
| `CLIENT_ID` | Yes | Discord application/client ID |
| `OWNER_ID` | Yes | Your Discord user ID (dashboard access) |
| `DISCORD_CLIENT_SECRET` | Yes | OAuth2 client secret (for dashboard login) |

### 2. Discord Developer Portal

1. Go to [discord.com/developers/applications](https://discord.com/developers/applications)
2. Select your application ‚Üí **OAuth2**
3. Under **Redirects**, add:
   ```
   https://<your-domain>/dashboard/auth/callback
   ```
4. Save changes

### 3. First-time Server Setup

Run `/setup` in your Discord server to configure log channels and roles before using any moderation commands.

## Dashboard

Visit `/dashboard` in a browser to access the admin panel. You'll be redirected to Discord login ‚Äî only the account matching `OWNER_ID` is granted access.

From the dashboard you can:
- Create and manage application forms
- Configure which Discord channel submissions are posted to
- Set accept/deny roles per server
- View and manage all submissions

### Application Flow

1. Create a form in the dashboard and set a target channel + accept/deny roles
2. Share the public apply link: `/apply/<formId>`
3. Applicants fill out the form ‚Äî a card with their answers is posted in your chosen channel
4. Staff click **Accept** or **Deny** on the Discord card
5. The applicant is automatically assigned the correct role and the card updates

## Notes

- The `node:sqlite` module emits an `ExperimentalWarning` on startup ‚Äî this is expected and harmless
- Slash commands are registered globally ‚Äî changes can take up to 1 hour to propagate
- Commands like `/warns`, `/messages`, `/balance`, and `/snipe` are intentionally open to all users and excluded from the role permission system
- Admins always bypass role permission checks
